## 3. 각 항목 상세

### 3.1 Last-note restoration

**Rust 측**
- `src-tauri/src/services/workspace.rs` (신규): 볼트별 머신 로컬 상태
  ```rust
  #[derive(Serialize, Deserialize)]
  #[serde(tag = "kind")]
  pub enum LastOpened {
      #[serde(rename_all = "camelCase")]
      Note { rel_path: String },
      #[serde(rename_all = "camelCase")]
      Journal { date: String },
  }
  ```
  → variant 이름은 PascalCase 유지(`Note`, `Journal`), 필드만 camelCase(`relPath`).
  처음에 enum 레벨 `rename_all = "camelCase"`로 시도했더니 variant까지
  소문자화되어 직렬화 테스트가 깨졌음. variant-level rename_all로 분리해 해결.
- `src-tauri/src/commands/workspace.rs` (신규): `workspace_get` /
  `workspace_set_last_opened` IPC.
- `src-tauri/src/services/git.rs`: `.naiteh/workspace.json`을 ensure_gitignore body에 추가.
- `src-tauri/src/lib.rs`: 두 핸들러 등록.

**TS 측**
- `src/lib/types.ts`: `LastOpened`, `WorkspaceState` 미러 타입 추가.
- `src/lib/api/workspace.ts` (신규): `workspaceGet`, `workspaceSetLastOpened` 래퍼.
- `src/shell/AppShell.tsx`:
  - 마운트 시 `workspaceGet` → `lastOpened`가 있고 에디터가 비어 있으면
    `openByRelPath`로 자동 오픈. 실패하면 stale 마커 제거.
  - `editorStore.subscribe`로 `open.key` 전이를 감지해
    `workspaceSetLastOpened` 호출. 닫힘(`null`)이나 볼트 전환 시에는
    이전 볼트 상태를 보존하기 위해 쓰지 않음.

### 3.2 Sync status in StatusBar

- `src/state/syncStore.ts` (신규): 최신 `SyncStatus` + `notInitialized` 플래그
  보유. `refresh()`는 `syncStatus` IPC를 호출하고, `NotFound` 예외면
  `notInitialized = true`로 잡음.
- `src/lib/syncStatusLabel.ts` (신규): `describeSyncStatus(status, notInit, nowMs)` 순수 함수.
  - not-initialised → `"Sync: off"`
  - null + 플래그 없음 → `"Sync: —"`
  - dirty → `"Sync: pending"` (항상 우선)
  - lastSync 없음 + remote 없음 → `"Sync: local only"`
  - lastSync 없음 + remote 있음 → `"Sync: never"`
  - 그 외 → `"Sync: 5m ago"` / `"Sync: just now"` (5초 이하)
- `src/lib/__tests__/syncStatusLabel.test.ts` (신규): 7개 케이스.
- `src/shell/StatusBar.tsx`: `<span>Sync: —</span>` → `<button>` (linkLike)
  로 교체, 클릭 시 `setViewMode("sync")`.
- `src/shell/StatusBar.module.css`: `.linkLike` 스타일(투명 버튼) 추가.
- `src/shell/AppShell.tsx`: 마운트 시 `refresh()` 1회 + 30초 간격
  `setInterval`. 볼트 전환 시 AppShell이 key로 재마운트되므로 인터벌도 자동 재시작.

### 3.3 Tags multi-select

`src/features/tags/TagsListPanel.tsx`를 광범위하게 리팩터링:

```tsx
const [selectedTags, setSelectedTags] = useState<Set<string>>(new Set());
const [mode, setMode] = useState<Mode>("any");
const [perTagNotes, setPerTagNotes] = useState<Map<string, NoteMeta[]>>(new Map());
const fetchedTagsRef = useRef<Set<string>>(new Set());
```

- `combine(perTagNotes, mode)`: relPath identity 기준으로 합집합/교집합 계산.
- Any/All 토글은 2개 이상 선택했을 때만 노출.
- Clear 버튼으로 선택 비우기.
- 태그 칩을 다시 클릭하면 선택 해제.

**버그 두 번 잡음**

1. **무한 렌더 루프**: 처음에 effect deps에 `[selectedTags, perTagNotes]`
   를 넣고 effect 안에서 `setPerTagNotes(new Map())`을 호출했더니
   매번 새 Map 레퍼런스가 생겨 무한 루프. Worker가 580초 타임아웃 후
   비정상 종료, 12개 테스트 hang. → deps에서 `perTagNotes` 제거.
2. **setState updater의 비동기 동작으로 IPC 미발사**: 두 번째 시도에서
   `toFetch.push()`를 `setPerTagNotes((prev) => {...})` 콜백 안에서
   하고 콜백 바깥에서 `toFetch.length === 0` 체크를 했더니, updater가
   비동기로 호출되어 항상 빈 배열로 보였음. 6개 테스트 실패. →
   `useRef<Set<string>>`로 fetched 마크를 동기적으로 관리하여 해결.

`TagsListPanel.module.css`에 `modeToggle`, `modeOption(+Active)`,
`clearButton` 스타일 추가. 테스트는 멀티셀렉트/모드 토글/Clear/디셀렉트
5개 케이스 신규 추가.
