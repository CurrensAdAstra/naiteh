## 1. 큰 그림

`naiteh`는 Tauri v2 + React 18 + TypeScript로 만든 로컬 우선 마크다운 노트/저널 앱입니다. 이 기간의 작업은 크게 네 갈래로 분류됩니다:

1. **신규 기능**: Evernote import, 이미지 paste/drop, Notes 폴더 관리, 로컬 Ollama, 네이티브 메뉴+단축키
2. **보안/안정성 강화**: 인증 토큰화, Argon2, vault mutex, conflict UI, symlink 차단, CSP, 첨부 size cap, audit-log rotation
3. **성능/캐시**: tag·timeline 인덱스, 쓰기 성공 시에만 invalidate
4. **구조/유지보수**: 에러 타입 세분화, naming 유틸 통합, domain 분할, store 결합 정리, 문서 재정합

## 4. 주요 설계 결정

### 보안 모델 (최종 형태)

- **인증**: `auth_login`이 `LoginResult { token, session }` 반환. 토큰은 256-bit hex random (`getrandom`). `SessionStore`가 Tauri-managed state에 인메모리 보관. 모든 admin IPC는 토큰을 받고 `require_admin`이 resolve. 재시작 시 모든 세션 무효.
- **비밀번호**: Argon2id + per-user salt (PHC 문자열 포맷). 레거시 SHA-256 해시는 다음 로그인 성공 시 자동 재해싱.
- **시드 계정**: `admin/admin` 하나만 (이전 `mgkyung` 하드코딩 제거).
- **경로**: `notes::resolve_in_vault`가 deepest existing ancestor를 canonicalize해서 vault root 포함 여부 검증 → 동기된 remote의 악성 symlink로 외부 접근 차단.
- **CSP**: `default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; ...` (CodeMirror의 runtime StyleModule 때문에 style만 unsafe-inline).
- **첨부**: 50 MiB cap (path와 bytes 양쪽). 클라이언트도 paste 전에 체크해 JSON 직렬화 절벽 차단.

### 동시성

- **Per-vault mutex** (`services/vault_lock.rs`): canonical vault root 키. 모든 mutating IPC가 `.lock().await` 후 작업. Tauri-managed state.
- **Tag 인덱스** (`services/index.rs`): `Arc<TagSnapshot>` 인메모리 캐시. 쓰기 성공 시에만 invalidate. timeline/activity도 같은 인덱스에서 빌드.
- **Sync pull**: dirty tree면 `AppError::Conflict`로 거부 (force-checkout으로 사용자 작업 덮어쓰기 방지). 충돌 발생 시 `<file>.conflict-<ts>.md` sidecar 작성 → Sync 패널의 Conflicts 섹션에서 keep-mine/keep-theirs 해결.

### AI Assist (로컬 우선 옵션)

- 키 optional (`with_optional_auth`). 로컬 엔드포인트 (`localhost`/`127.0.0.1`/`::1`) 감지 → 키 없이도 "ready".
- Ollama: `ollama serve` + `ollama pull <model>` 후 Settings에서 "Use local Ollama" 프리셋 → "Load models" → 선택 → Save.
- `ai_list_models`는 OpenAI 호환 `GET {base}/models`로 둘 다 동작.
